Linting

¿QUÉ ES LINTING?


Linting es un término usado para designar herramientas que su función principal es revisar minuciosamente el código creado para poder encontrar diferentes tipos de errores. Se sabe que en la codificación es algo inevitable que el código creado este perfecto, casi siempre presenta diferentes errores, ya sean errores de sintaxis, códigos poco intuitivos, fallas en ejecución, entre otros, algunos pueden ser graves y otros un poco más sencillos y es por ello que siempre hay que captar a tiempo estos errores. Linting es de gran ayuda, facilita el trabajo de estar buscando en donde es exactamente que se genera el fallo del código, busca los errores que puedan estar afectando al código y algunos de los linteres incluso pueden dar ciertas sugerencias de cómo darle solución al error o también llegan al punto de que los pueden arreglar.

Los lenguajes como Javascript y Python se benefician mucho gracias al linting ya que estos lenguajes no presentan una fase de compilación para mostrar errores antes de una ejecución.

Las herramientas linting son del grupo de programas de herramientas de análisis estático, que quiere decir que es un proceso de revisión sin ejecutar el programa, se realiza sobre el código fuente o alguna clase de código objeto.

¿Cuál es el significado de lint?

El significado de lint originalmente es el nombre dado a un programa en particular que marca las fallas en la construcción de código fuente del lenguaje C. Actualmente el termino se aplica para aquellas herramientas que su función principal es eso, poder identificar errores en el código de manera rápida y sencilla.



¿CÓMO SE UTILIZA EN JAVASCRIPT?


Un ejemplo de herramienta linting es ESLint, que básicamente es una herramienta de código abierto enfocado en el proceso de “linting” para JavaScript, dado que este es un lenguaje dinámico y de tipado débil, es muy fácil caer o crear errores a la hora de trabajar en la creación de código. Cuanddo ESlint encuentra algún error o bien lo repara de forma automática o advierte del error a través de distintos mensajes, basándose en un conjunto de reglas definidas y configurables. ESLint utiliza un sistema de reglas que permiten definir qué es y que no es posible dentro del código. ESLint está escrito en NODEjs y es posible instalarlo desde npm.


“ESLint es conocido como uno de los linters más populares para Javascript”

Características principales de ESLint:


✔ Puede mostrar errores de sintaxis

✔ Muestra los errores cuando no sigue un buen manejo del código

✔ Brinda diferentes tipos de sugerencias o recomendaciones para el código

✔ Si el error esta al alcance puede solucionarlo él mismo.

✔ Ayuda a llevar un mejor rumbo del código en otras palabras nos ayuda a mantenerlo claro.
